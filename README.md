# 🎧 Telegram Bot v2 — модульная архитектура

## 📁 Структура проекта

```
telegram_bot/
├── bot.py              — точка входа, собирает роутеры
├── config.py           — переменные окружения
├── database.py         — вся работа с db.json (статистика, коды, пользователи, рефералы)
├── keyboards.py         — все inline-клавиатуры
├── states.py            — FSM состояния
├── handlers/
│   ├── user.py          — /start, подписка, ввод ключа, реферальная программа
│   ├── admin.py         — /admin, статистика, коды
│   └── broadcast.py      — рассылка всем пользователям
├── requirements.txt
└── .env.example
```

## ⚙️ Установка

```bash
pip install -r requirements.txt
cp .env.example .env
# заполни .env своими данными
python bot.py
```

**Railway:** переменные добавляются через Settings → Variables, а не через `.env`.

## ✨ Что нового

### 1. Кнопки "Примеры работ" и "Заказать"

Теперь это **прямые url-кнопки** — открывают ресурс в 1 клик, без промежуточного экрана.

Технический момент: Telegram не позволяет одной inline-кнопке одновременно открывать ссылку и отправлять callback на сервер бота, поэтому точное "нажал именно эту кнопку" технически не отследить. Вместо этого статистика считает **переход на страницу 2** (после успешной проверки подписки) как событие `referral_join` — это даёт честную метрику конверсии "подписался → дошёл до меню с предложениями", что для оценки эффективности воронки даже полезнее, чем клик по конкретной кнопке.

### 2. Рассылка (📢 в админке)

Процесс:
1. Нажимаешь **«Рассылка»**
2. Присылаешь любое количество сообщений подряд — текст, фото, видео, файлы, ссылки, в любом порядке и комбинации
3. Нажимаешь **«Готово»**
4. Видишь подтверждение: сколько сообщений и скольким пользователям отправится
5. Жмёшь **«Отправить всем»**
6. Бот шлёт всем зарегистрированным пользователям с защитой от лимитов Telegram (~20 сообщений/сек), пропускает заблокировавших бота, в конце показывает отчёт: доставлено / не удалось

### 3. Реферальная система

- Каждый пользователь получает уникальную ссылку вида `https://t.me/your_bot?start=ref123456789`
- Кнопка **«🤝 Реферальная программа»** на странице 2 показывает ссылку, количество приглашённых и прогресс до бонуса
- Порог бонуса настраивается через `REFERRAL_BONUS_THRESHOLD` в `.env` (по умолчанию 3 друга)
- В админке — **«🏆 Топ рефералов»**: рейтинг самых активных пользователей
- **Важно:** бот сам не выдаёт награду автоматически — выдай её вручную, когда пользователь напишет "выполнил условие". Если нужна автоматическая выдача кода-бонуса при достижении порога, могу добавить.

## 📊 Как теперь считается статистика

Раньше считались клики (повторные нажатия от одного юзера = +1 каждый раз).
Теперь считаются **уникальные пользователи в день** — если человек нажал кнопку 5 раз за день, это всё равно 1 в статистике за сегодня.

## 🚀 Идеи для повышения конверсии

Помимо того что уже реализовано (реферальная система, рассылка), вот что технически реально добавить:

**Автовыдача бонуса по рефералам** — как только у пользователя счётчик приглашённых достигает порога, бот сам присылает ему секретный код или сообщение, без участия админа. Это сильно повышает виральность, потому что человек получает награду мгновенно, а не ждёт ответа от тебя.

**Триггерные сообщения** — если пользователь подписался, но за N часов не ввёл ключ и не написал "заказать", бот сам присылает напоминание ("Не забыл про скидку на сведение?"). Технически реализуется через задачу `asyncio` с проверкой `users` раз в час.

**A/B тест текста главного меню** — двум случайным половинам аудитории показывать разные формулировки кнопок и сравнивать, у кого выше доля дошедших до "Заказать". Можно встроить в текущую архитектуру без больших изменений, просто храня `variant` в профиле пользователя.

**Воронка с отложенным контентом** — вместо немедленной выдачи всего через код, разбить ценный контент на "капельную" рассылку: день 1 — один пример работы, день 3 — следующий, день 5 — призыв заказать. Это держит аудиторию вовлечённой дольше, чем разовая выдача.

**Ограничение по времени на код** — коды можно сделать одноразовыми (использован → удаляется автоматически) или с сроком действия, чтобы создать ощущение дефицита и подталкивать к быстрому действию.

Если что-то из этого интересно — могу реализовать в этой же архитектуре, дополнительные модули вписываются без переписывания существующего кода.
